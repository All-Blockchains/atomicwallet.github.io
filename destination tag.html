Destination tag

Preface

Management of Ripple (XRP), Stellar (XLM) and Monero (XRM) has some significant features when compared to the standard cryptocurrencies management. To have a full-control over these assets you need to get acquainted with their unique characteristics.

These assets are using Destination Tag technology, which purpose is to define the specific recipient of the transaction.

How Destination tag works?

For example, some trader can give you his XRP address to send, and you’ll notice that you have the same XRP address on the exchange service. It happens because the exchange can use a one XRP address for many traders on the same platform. The Destination Tag is different for each user and exactly establishes the recipient of the assets.

It is very important to have a complete understanding of Destination Tag, because if you’ll set there random numbers or unsuitable values instead of the Tag  — you’ll lose your funds, they will be sent to any other user or on the main exchange address without the Destination Tag.

The Destination Tag purpose is identificate transaction recipient.
It’s like the numbers of rooms. If you are living in the apartment building you’ll need to use exactly your room number. If you are living in a private house you’ll not.

How to use Destination Tag?

The Destination Tag is not needed if you have your own personal address of the coin, as like in the decentralized Atomic Wallet — every user has they own coin address. But when you manage coins with centralized exchange — Destination Tag is necessary requires correct assets exchange.

Don’t use your exchange service username as a Destination Tag! It will be unsuitable values and assets will be sent to the main exchange address without the specific appointment to the recipient.

  • In Ripple Destination Tag can be shown in the address after the «?dt=». As like as: «rw7AxoYRb5MyRf3oRjSVPtgsQYQLp6rqGn?dt=1234567890»
  • Stellar is called the Destination Tag as «Memo ID»
  • Monero is called the Destination Tag as «Payment ID»

It’s just the different names of the same feature.
